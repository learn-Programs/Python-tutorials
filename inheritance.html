Inheritance

In this article, you will learn about inheritance in Python.

Inheritance is an important feature of the object oriented programming language. Inheritance provides code reusability to the program because inheritance enable us to share functionality between classes.

What is Inheritance?

In object oriented programming, Inheritance is a mechanism in which a new class is derived from an already defined class.

The derived class is known as a subclass or a child class. and the class from new class is derived is called as base class or superclass or parent class.

Syntax:

class BaseClass:
    base class block
    
class DerivedClass(BaseClass):
    derived class block

The derived class inherits all the methods and properties of the base class. The derived class can also has its own methods. 

class Animal:

    def __init__(self, name, color):
        self.name = name
        self.color= color

class Dog(Animal):

    def bark(self):
        print("Woof!")

#object
moti = Dog("Moti", "Brown")

print(moti.color)

moti.bark()

Output 

Brown
Woof!

Method Overriding

The subclass can also replace methods of the superclass. The process of replacing methods defined in super class with new methods with the same name in the derived class is known as Method Overriding.

When base class and derived class has methods with same name, the method in derived class overrides that method in the base class. which means that the whenever we call that method, method in derived class gets executed.

To avoid this we can use super() method in derived class. 

class Animal:

    def speak(self):
        print("Speaking")

class Dog(Animal):

    def speak(self):
        print("Barking")
        super().speak()

d = Dog()
d.speak()

Output 

Barking
Speaking

Python has two built-in functions isinstance() and issubclass() are used to check inheritances.

Function isinstance() returns True, if the object is an instance of the class. each and every class in Python inherits from the base class object. 

>>> isinstance(d, Dog)
True

>>> isinstance(d, Animal)
True

>>> issubclass(Dog, Animal)
True

Private Variables

“Private” variables are those that cannot be accessed from outside the class. These attributes, may it be data or methods, can only be accessed from inside an object.

These types of members don’t exist in Python. However, Python provide a limiled support for such a mechanism, called name mangling. Where a name prefixed with two leading underscores and no more than one trailing underscore will be considered as a private member.

for example, __spam is considered as private member.

Name mangled methods can still be accessed externally, but by a different name. The method __privatemethod of class Spam could be accessed externally with _Spam__privatemethod. 

class Spam:

    __egg = 7

    def print_egg(self):
        print(self.__egg)


s = Spam()

s.print_egg()

print(s._Spam__egg)
print(s.__egg)

Output

7
7
AttributeError: 'Spam' object has no attribute '__egg'
