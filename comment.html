Python Comments

In this article, we will learn about Python comments and why we should use comments in our program.

Python interpreter typically tries to run all of the source code that we put into our source code window, but sometimes it can be helpful to leave natural language notes or explanations for ourselves or for other programmers looking at the source code. In order to do that, we have what are called comments. Comments are used to explain code, when the basic code itself isnâ€™t clear. Although comments do not change the outcome of a program, they still play an important role in any programming language and not just Python.

Comments are helpful when someone else has written code and you are analyzing it or making changes, or even when we go through our own old written program, by reading comments you can understand the purpose of code much faster than just going through the actual code.

In Python, we write a comment by using the # symbol and writing what we want after it. when we write comment, then Python ignores everything that comes after the # symbol and what comments do are they tell the Python interpreter to ignore these portions of the source code, and only to run whats not commented.   • Single-line comment
  • Multi-line comment
  • In-line comment type
