Exception Handling

In this tutorial, we will learn Python Errors and Exception Handling.

There are two types of errors in Python programming languages.

  1. Syntax Error
  2. Exceptions.

Syntax Error

The Syntax Errors occurs when there is a mistake in syntax, which means that when do not follow the proper structure of the language.
The Syntax Errors are also called as ‘Parsing Error’. 

if True
print("Hello World")

Output 

File "<stdin>", line 1
    if True
           ^
SyntaxError: invalid syntax

Since colon (:) is missing after if statement SyntaxError occurs. Filename and line number are printed so could know where to look.

Exceptions

The Errors that occur during the execution of the program is called Exceptions. Whenever these type of runtime error occur, Python creates an exception object. It prints traceback to that error along with some details about why that error occurred. 

>>> 1 / 0

Traceback (most recent call last):
File "<stdin>", line 1, in <module>
     1/0
ZeroDivisionError: division by zero

The last line of the traceback output message tell us about, what type of exception was reaised and some relevant information about that exception. Exceptions come in different types, and the type is printed as part of message.

In the above example; exception ZeroDivisionError with message as division by zero occur, which means that the second argument/operand used in a divison operation was zero.

There are various built-in exceptions in Python that are raised when corresponding error occurs. we can view all built-in exceptions using the local() built-in functions as follows 

>>> locals`()['__builtins__]`

Exception Handling

It is possible to write a program that handles selected exceptions. This can be done using try...except statement.

The try statement specifies exception handlers and cleans up the code and except clause specify one or more exception handlers.

If the program contains a code that may raise an exception is placed inside the try clause and the code that handles exception is written in except clause.

Syntax: 

try:
    run this code
except:
    run this block if an exception occurs


# ZerodivisionError

try:
    a = int(input('a = '))
    b = int(input('b = '))
    print(a/b)

except ZeroDivisionError:
    print('Enter the number greater than zero')

Output

a = 1
b = 0
Enter the number greater than zero

A try statement may have more than one except clause. An except clause may have multiple exceptions name parenthesized in a tuple.

Syntax: 

... except (RuntimeError, TypeError, NameError):
...    pass

Raising Exception

In Python programming, exceptions are raised when corresponding errors occur at run time, but we can forcefully raise it using the keyword raise. 

try:

    a = int(input("enter the positive number: "))

    if a <= 0:
        raise ValueError("This is not a positive number!")

except ValueError as value:
    print(value)

Output

enter the positive number: -1
This is not a positive number!
