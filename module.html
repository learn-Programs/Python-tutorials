Python Modules

In this tutorial, you will learn, How to create, import modules in Python.

What are Modules?

A module is a file containing Python definitions and statements. We can say that, a module is nothing but a Python source code file that we save with .py extension.
The file name is module name with the suffix .py .

To Understand more about modules, Let's create a following Program that has function called sqr(num) which takes number, and returns a square of that number, and save that file as square.py.

Here, square.py is called module and square is module name.

example.py

#'square.py'

def sqr(num):
    return num * num
    
num = 2
sqr(num) # function call

We use modules to break down large programs into small manageable and orgranized files. Modules provides reusablitilies of code.

We can defined our most used functions in a module and import it, instead of copying its definition into each program every time.

How to import modules in Python?

We can import the definition inside a module, into another module or the interactive interpreter in Python.

To import module, we can use import keyword followed by the name of the module.

Now, lets enter into the Python interpreter and import the module square.py which we created earlier.

example.py

>>> import square

This does not enter the names of functions defined in square module directly in the current symbol table; it only enter the module name square there.

We can access the functions and definitions inside the module using module name with . dot operator as follows.

example.py

>>> square.sqr(2)
>>> 4

Python provides a library of standard modules, which we can use in our program in similar way as we did before using import.

Each module has it's own private symbol table, which is used as the global symbol table by all functions defined inside that module. Thus, the creator of module can use global variables in the module without worring about the clashes with a user's global variables.

Python "from .. import" statement

We can import the specific names / functions from module without importing whole module.

example.py

>>> from square import sqr
>>> sqr(2)
4

In the above example, we imported only sqr definition/function from a square module.

In this case, we don't need use . dot operator, we can used it as it directly. we can also import attributes of a module in similar mannar as above.

example.py

>>> from square import num
>>> num
2

This does not introduce the module name from which the imports ( definitions/ functions/ attributes ) are taken in the local symbol table. so the square is not defined here.

import all names

We can import all names that a modules defines, using * operator.

example.py

>>> from square import *
>>> sqr(3)
9
>>> num
2

This import all names(definitions) from the modules defined except those beginning with an underscore (__).

Although, it is not good practice of include importing * from a module, since it often causes poorly readable code.

importing with renaming

We can rename module while importing it into our program using as keyword as follows:

example.py

>>> import square as s
>>> s.sqr(4) #accessing function in s module using '.' operator.
16

This is a same way importing module with import square , only difference is that, it is now available as s.

Now, square is not recognized in our scope, since we change the name as s. Hence, the square.sqr(4) is invalid here.

It can also be used using from with similar effects as follows:

example.py

>>> from square import sqr as sq
>>> sq(5)
25

Python module search path

When module is imported, the interpreter first search for a built-in module with that name. If file not found, then it searches that file in a list of directories given by the variable sys.path.

sys.path is initialized from these locations listed below

  • The directory containing the input script or current directory.
  • PYTHONPATH ; a list of directory names.
  • The installation-dependent default directory.

After initialization, Python programs can modify sys.path.

Reloading Modules

A module can contain executable statements as well as functions definition.
These statements are intended to initialized the module. They are executed only the first time the module name is encountered in an import statement.
Let's create a test.py module.

example.py

#test.py
print("test.py module imported.")

def function():
    print("Welcome to function of test module.")

Now, let's try to import it more than one times in Python interpreter.

example.py

>>> import test
test.py module imported.

>>> import test

>>> import test

Did you notice?, when we import the module test first time, only then print statement is get executed. Which means that our modules was imported only once.

Now, if our module changed while running program, then we must restart the interpreter which is not a good idea.

But, Python provide an efficient way to do it using importlib.reload() method.

example.py

>>> import test
test.py module imported.

>>> import test

>>> import importlib
>>> importlib.relaod(test)
test.py module imported

dir() function

A module can be imported, when imported, module is evaluaed, and a module object is created. That module object has attributes.

The dir() function is built-in function which is used to find out that objects of the modules when module is evaluated.

It retruns a sorted list of strings which contains a all types of names such as variables, functions, modules, etc.
for example:

example.py

>>> import square

>>> dir(square)
['__builtins__','__doc__', '__file__', '__name__', 'num', 'sqr']

Here, we can see a sorted list of names - along with `num` and `sqr` . All other names that begin with double underscore (__) are default Python attributes associated with module, when module was executed.

  • __doc__ : the doc string of the module.
  • __name__ : The name of the module when the module is imported.

example.py

>>> import  square
>>> square.__name__
'square'

We can find list of all the names that we defined in our current namespace using dir() without any argument.
